The user wants to build a puzzle mobile application for children aged 5-8.

**Core Requirements:**
- **Image Source:** Admins (and later, users) can upload pictures to be turned into puzzles.
- **Gameplay:** Children select a pre-uploaded puzzle from a gallery and choose a difficulty level. The game involves assembling the shuffled puzzle pieces.
- **Features:** The app should include a timer, hints, move counter, and a celebration animation upon completion.
- **Scoreboard:** Maintain the last 10 scores for each puzzle and display them to the player.
- **Analytics:** The app creator needs a separate web dashboard to track core usage metrics, engagement, and technical performance.

**Latest User Request (High Priority):**
1.  **Architecture Change (Performance & Privacy):** User-uploaded images must be stored locally on the device. Only pre-loaded images should be fetched from the server.
2.  **Pre-loaded Content:** Add pre-loaded images from the server, organized into categories like 'flags', 'animals', 'alphabets'.
3.  **UI/UX Changes:**
    - On the Choose a Puzzle screen, move the image library to the top and the Upload feature to the bottom.
    - Make the animation on the home screen clickable, navigating to the puzzle gallery.
4.  **Upload Flow:** When users upload a picture, allow them to assign it to a category.
